# LeLe Download 优化总览

> 优化对象：Tauri 2.x 桌面下载器（Rust 后端 + React + Vite + TailwindCSS）
> 优化方式：按《项目分析报告》依次修复 7 项缺陷，全部为真实可用的工程修复。

## 已完成的优化（Fix 1–7）

### Fix 1 · 浏览器扩展集成（HTTP 桥鉴权）
- `src-tauri/src/main.rs`：HTTP token 由"始终生成并强制校验"改为 `Option<String>`——仅在显式设置环境变量 `LELE_HTTP_TOKEN` 时才校验；否则本地回环 `127.0.0.1` 免鉴权直接放行。
- `extension/background.js`：请求地址由 `http://localhost:45678` 改为 `http://127.0.0.1:45678`，避免 IPv6 `localhost` 解析失败。
- 效果：浏览器扩展"发送下载"不再 401，端到端打通。

### Fix 2 · 启动恢复历史
- `main.rs` `setup`：原 `load_tasks()` 仅打日志。现改为：先把被中断（Downloading/Pending）的任务标记为 Failed 写回 DB，再为每个已存在任务 `restore_task()` 注入 `DownloadManager`，重启后历史可见、可续。

### Fix 3 · 设置面板后端接线
- `src/components/SettingsPanel.tsx`：`handleSave` 改为 `async`，调用 `invoke('update_download_settings', { settings: { maxThreads, maxConcurrent, speedLimitKb, ... } })`，同时落 localStorage。
- `src-tauri/src/commands.rs`：`DownloadSettings` 增加 `#[serde(rename_all = "camelCase")]` 与 `speed_limit_kb` 字段（并补全 `Default` 实现，避免编译缺失字段）。

### Fix 4 · 真正的断点续传
- 单线程 `download_single`：发送 `Range` 后校验响应状态——**仅当服务端返回 `206 Partial Content` 才以 append 模式续传**；若服务端忽略 Range 返回 `200`，则重置 `resume_from = 0` 当作全新下载（避免 append 导致文件损坏/内容重复）。
- 分片 `download_chunk_to_temp`：每个分片按 `start+resume_from` 续传；新增 **206 强校验**——若分片请求未拿到部分响应则显式报错，杜绝"每个分片写入整份文件→合并损坏"的静默灾难。

### Fix 5 · 并发限制 + 速度限制（原 no-op）
- 并发：用 `Arc<AtomicUsize>` + `tokio::sync::Notify` 实现动态并发闸门（替代固定 `Semaphore`），`max_concurrent` 可在设置中运行时变更即时生效。
- 速度：`RateLimiter`（共享令牌桶）按 `speed_limit_kb` 限流。同一任务所有分片共享一把限速器，聚合速率受控；`0` 表示不限速。`download_single` 与 `download_chunk_to_temp` 的每段数据在写入前调用 `acquire()` 申请额度。

### Fix 6 · 打包目标统一
- `src-tauri/tauri.conf.json`：`bundle.targets` 由 `["nsis"]` 改为 `"app"`，匹配可用的安装包构建路径。

### Fix 7 · 安全与清理
- `tauri.conf.json`：`security.csp` 由 `null` 改为真实 CSP 策略串。
- `download.rs`：新增 `sanitize_filename()`（剥离路径分隔符、Windows 非法字符、控制字符、首尾 `.`，限长 200），`extract_filename` 经其清洗，防路径穿越。
- `open_folder`：移除过度校验（原拒绝 `(`、`)`、`$`、`` ` ``、`|`、`&`、`;`），仅拦截 `".."` 路径穿越（`explorer.exe` 参数本身 shell 安全）。
- `remove_download` / `cancel_download`：调用 `cleanup_task_files()` 清理残留 `.part.*` 临时分片。

## 本轮额外修复（健壮性 / 编译正确性）
- **进度条冻结回归修复**：分片下载的"共享进度计数器"此前在循环内被每个分片重声明为独立计数器、进度上报任务又另起一把，导致进度停滞在跳过字节处。现已统一为循环前声明的单一 `Arc<AtomicU64>`，各分片 `fetch_add` 累加，上报任务读取同一把。
- **编译阻断修复**：`DownloadSettings::default()` 漏初始化 `speed_limit_kb` 字段，已补 `speed_limit_kb: 0`。

## ⚠️ 构建须知
- 当前环境**无 Rust 工具链（cargo / rustc 均不可用）**，上述 Rust 改动**未经编译验证**。仅前端 `tsc` 此前通过。
- 请在装有 Rust 的环境执行 `cargo check` / `cargo build` 后再发布。
- 安全提示：Fix 1 默认放开本地回环免鉴权以打通扩展集成；若需更强隔离，可设置 `LELE_HTTP_TOKEN` 启用 token 校验（扩展侧需同步携带 `X-Auth-Token`）。
